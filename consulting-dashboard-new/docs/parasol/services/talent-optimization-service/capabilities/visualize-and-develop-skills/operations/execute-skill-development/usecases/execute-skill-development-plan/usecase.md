# ユースケース: スキル育成計画を実行する

## 基本情報
- **ユースケースID**: UC-SKILL-04
- **アクター**: リソースマネージャー、メンバー、チームリーダー、研修担当者
- **概要**: 策定されたスキル育成計画を実際に実行し、メンバーのスキル向上と組織能力強化を継続的に推進する

## 事前条件
- スキル育成計画が策定されている（SkillDevelopmentPlan.status = "approved"）
- 対象メンバーの合意が得られている
- 育成リソースが確保されている
- 実行権限が確認されている
- 進捗測定方法が定義されている

## 事後条件
### 成功時
- スキル育成プログラムが実行されている（SkillDevelopmentExecution.status = "in_progress"）
- 進捗追跡が開始されている
- メンバーへの通知が配信されている
- 育成履歴が記録されている
- 次回評価スケジュールが設定されている

### 失敗時
- 実行失敗の詳細理由がログに記録されている
- 代替実行方法が提案されている
- メンバーへの説明と調整が実施されている
- リソース調達の課題が特定されている

## 基本フロー
1. **リソースマネージャーがスキル育成計画の実行を開始する**
   - 育成計画の最終確認・対象メンバーとの調整
   - 実行スケジュールの確定・リソース配分の確認

2. **システムが育成実行環境と監視機能を提供する**
   - → UC-AUTH-02: スキル育成実行権限を検証する
   - 育成プラットフォームの準備・進捗追跡システムの初期化
   - 学習リソースへのアクセス権設定

3. **システムが個別化育成プログラムを開始する**
   - メンバー別カスタマイズ・学習パス最適化
   - 進捗モニタリング・自動リマインダー設定

4. **システムが育成活動の進行管理を実行する**
   - 学習進捗の自動追跡・成果測定
   - フィードバック収集・調整提案

5. **システムが育成結果の記録と評価を完了する**
   - 育成成果の記録・スキルレベル更新
   - → UC-NOTIFY-01: 育成進捗通知を配信する
   - → UC-AUTH-03: 育成実行ログを記録する

## 代替フロー
### 代替フロー1: 段階的実行
- 3a. 大規模育成で段階的実行が必要な場合
  - 3a1. 優先グループの先行実行・パイロット実施
  - 3a2. 段階的展開・フィードバック反映
  - 3a3. 基本フロー4（進行管理）に進む

### 代替フロー2: 緊急スキル強化
- 3b. 緊急性の高いスキル強化が必要な場合
  - 3b1. 集中プログラムの実行・優先リソース配分
  - 3b2. 短期集中学習・専門指導・成果検証
  - 3b3. 基本フロー4に戻る

## 例外フロー
### 例外1: メンバー参加困難
- *a. メンバーが育成に参加できない場合
  - *a1. → UC-AUTH-03: 参加困難ログを記録する
  - *a2. 代替スケジュール・個別対応・リモート学習
  - *a3. 調整完了後、基本フロー3に復帰

### 例外2: リソース不足
- *b. 育成リソースが不足した場合
  - *b1. リソース調達・代替手段検討
  - *b2. 外部研修・オンライン学習・メンタリング
  - *b3. リソース確保後、基本フロー3に復帰

### 例外3: 進捗遅延
- *c. 育成進捗が大幅に遅延した場合
  - *c1. 遅延原因分析・個別支援強化
  - *c2. スケジュール調整・追加サポート・目標見直し
  - *c3. 改善後、基本フロー4に復帰

## 特別要件
- **性能**: 育成開始処理5秒以内、進捗更新3秒以内、レポート生成10秒以内
- **可用性**: 育成システム稼働率99.5%以上、24時間365日利用可能
- **セキュリティ**: 育成データ暗号化、アクセス制御、個人情報保護、監査証跡
- **継続性**: 長期育成対応、進捗継続性、中断再開機能、データ保持
- **個別性**: メンバー個別対応、学習スタイル適応、カスタマイズ、柔軟性
