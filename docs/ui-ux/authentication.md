# 認証とロール管理

## 認証方式

- **メールアドレス/パスワード**: 基本的な認証方法
- **MFA対応**: 多要素認証に対応

## ロール定義

システムには以下の5つのロールが定義されています：

1. **Executive**: 経営層向け
2. **PM**: プロジェクトマネージャー向け
3. **Consultant**: コンサルタント向け
4. **Client**: クライアント向け
5. **Admin**: システム管理者向け

## アクセス制御

- **ロールベースアクセス制御（RBAC）**を採用
- 各ロールに応じて表示される機能やメニューが動的に変更される
- APIレベルでもロールに基づいたアクセス制御を実装
